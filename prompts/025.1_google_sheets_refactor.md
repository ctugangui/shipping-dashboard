# Prompt 25.1: Refactor Google Sheets Parsing Logic

**Role:** Backend Refactoring Engineer
**Context:** We successfully implemented the basic `GoogleSheetsService`. However, the real-world Google Sheet data is messy: tracking numbers can share a single cell separated by newlines, they often have quantity suffixes (like " x 9" or "x 1"), and the column position might move in the future.
**Goal:** Refactor the `getTrackingNumbers()` method in `GoogleSheetsService.ts` to dynamically find the correct column and aggressively clean the extracted text.

# Requirements

## 1. Dynamic Column Search
* **File:** `src/services/GoogleSheetsService.ts`
* **Action:** Update the Google Sheets API fetch call. Instead of fetching a hardcoded range (like `Sheet1!A:A`), fetch the entire sheet specified by `process.env.GOOGLE_SHEET_NAME`.
* **Logic:** * Read the first row (the headers). 
  * Find the array index that strictly matches `process.env.GOOGLE_SHEET_TRACKING_COLUMN`.
  * If the column is not found, throw an Error: `Tracking column not found in the sheet.`

## 2. Aggressive Data Cleaning
* **Action:** Loop through the remaining rows and grab the string at the found column index.
* **Logic:** For each cell value:
  1. Skip if the cell is undefined or empty.
  2. Split the cell string by newline (`\n`) to handle multiple numbers in one cell.
  3. Map over the split lines. For each line, use regex or string splitting to remove the quantity suffix. (e.g., `.replace(/ ?x ?\d+/gi, '').trim()`).
  4. Filter out any resulting empty strings.
* **Output:** The method should flatten all of this into a single, clean `string[]` array containing *only* the raw tracking numbers.

# Verification
1. Run `npm run build` to ensure no TypeScript errors.
2. The user will test clicking the "Sync from Sheets" button in the UI. 
3. The backend should now successfully parse complex cells like `"1LSCXXH02491347 x 9\n9200190383434315797222 x 1"` into two distinct tracking numbers, skip any existing ones, and track the new ones.
