# Prompt 028.1: Track123 v2.1 Auto-Registration Flow

**Role:** Backend API Engineer
**Context:** Our `Track123Service` is successfully hitting the query endpoint, but the API returns `"code":"A0400", "msg":"trackNo not registered"` for new OnTrac numbers. The official documentation states we must first import the number using the `v2.1` import endpoint.
**Goal:** Update `Track123Service.ts` to automatically register unknown tracking numbers using the exact v2.1 endpoints and payload shapes before querying them.

# Requirements

## 1. Explicit Import Logic (`src/services/Track123Service.ts`)
* **Action:** Refactor the tracking flow inside `getShipment(trackingNumber)`.
* **Step 1 (Query):** Make a `POST` request to `https://api.track123.com/gateway/open-api/tk/v2.1/track/query`. 
  * Headers: `accept: application/json`, `content-type: application/json`, and `Track123-Api-Secret`.
  * Body must be: `JSON.stringify({ trackNoInfos: [{ trackNo: trackingNumber }] })`.
* **Step 2 (Check Rejection):** If `data?.rejected?.[0]?.error?.code === 'A0400'`, trigger the registration flow:
  * Log to terminal: `console.log("[Track123Service] Registering unknown number " + trackingNumber);`
  * Make a `POST` request to `https://api.track123.com/gateway/open-api/tk/v2.1/track/import`.
  * **Crucial:** The body for the import endpoint must be a raw array: `JSON.stringify([{ trackNo: trackingNumber }])`.
  * Wait 2000ms: `await new Promise(resolve => setTimeout(resolve, 2000));`
  * Retry the Step 1 Query exactly as before.

## 2. Safe Fallback Parsing
* **Action:** If the second query *still* returns no content (because Track123 takes a few minutes to fetch the data from OnTrac in the background), do NOT throw an error. 
* **Fallback Return:** Return a valid `UnifiedShipment` object containing the `trackingNumber`, with the `carrier` set to `ONTRAC`, the `status` set to `PROCESSING`, and the `location` set to "Pending Track123 Update". This ensures the package makes it onto our Kanban board immediately so it can be updated in future syncs.

# Verification
1. Run `npm run build`.
2. The user will click "Sync from Sheets" in the UI.
3. The backend should log the "[Track123Service] Registering unknown number..." message, successfully hit the `v2.1/track/import` endpoint, and safely place the packages into the UI.
